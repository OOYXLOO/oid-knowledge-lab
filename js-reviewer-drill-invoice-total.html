Candidate code

Invoice total calculator

export function calculateInvoiceTotal(items, discountPercent = 0) {
  const subtotal = items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  const discount = subtotal * discountPercent / 100;
  return Math.round((subtotal - discount) * 100) / 100;
}

Review verdict

Revise before acceptance

The implementation is readable and handles the happy path, but it trusts every input. That is risky for customer-facing billing, invoice previews, and checkout flows because invalid values can produce NaN, negative totals, or accidental fractional quantities.

Findings

Issues I would raise

HighInvalid item values can produce NaN."abc" * 2 becomes NaN.Validate every price and quantity before calculating.
HighNegative values can create impossible invoices.Negative price or quantity can produce a negative or misleading total.Reject negative prices and quantities.
MediumDiscount is not bounded.A discount of 150 percent returns a negative total.Require discount from 0 to 100.

Focused cases

assert.equal(calculateInvoiceTotal([{ price: 10, quantity: 2 }]), 20);
assert.equal(calculateInvoiceTotal([{ price: 19.99, quantity: 2 }], 10), 35.98);
assert.throws(() => calculateInvoiceTotal(null), /items must be an array/);
assert.throws(() => calculateInvoiceTotal([{ price: "abc", quantity: 2 }]), /price/);
assert.throws(() => calculateInvoiceTotal([{ price: -1, quantity: 2 }]), /price/);
assert.throws(() => calculateInvoiceTotal([{ price: 10, quantity: -1 }]), /quantity/);
assert.throws(() => calculateInvoiceTotal([{ price: 10, quantity: 1.5 }]), /quantity/);
assert.throws(() => calculateInvoiceTotal([{ price: 10, quantity: 1 }], 150), /discount/);
