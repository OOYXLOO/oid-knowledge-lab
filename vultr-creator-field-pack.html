Proposed topic

Build a Static OID Evidence Dashboard on Vultr with Node.js and Object Storage

Short description

I would like to create a practical Vultr tutorial showing developers how to generate safe JSON and Markdown evidence artifacts with Node.js, publish a static review dashboard, add a dataset manifest, and keep credentials, private inventories, and copied third-party page bodies out of the release.

The topic is based on a working public proof project, OID Knowledge Lab, which already includes static pages, dataset manifests, source-policy notes, publish-boundary checks, and implementation-backed writing samples. If accepted, I would rewrite the final tutorial specifically for Vultr's product focus, editorial style, and technical review requirements.
