Short profile

Code evaluation approach

I treat generated code and model answers as untrusted drafts. I restate expected behavior, identify hidden assumptions, design small checks, compare candidate answers, and write concise verdicts that separate working behavior from unresolved risk.

Evaluation frame:
1. Restate the task and expected behavior.
2. Identify edge cases and unsafe assumptions.
3. Run or sketch focused tests.
4. Compare candidate answers against evidence.
5. Recommend accept, revise, or block with a narrow fix path.

Review sample

How I would answer a coding-evaluation prompt

StepAnswer shape
Task restatementClarify inputs, outputs, constraints, and failure modes before judging an implementation.
Evidence checksUse small examples, edge cases, parser boundaries, invalid inputs, and regression checks.
VerdictState whether the answer is correct, partially correct, or unsafe, then cite the exact reason.
RepairOffer the smallest useful fix and describe the test that should pass after the change.
Residual riskName any assumption not proven by the current evidence.

Public boundary

What this packet does not use

All linked materials are public project artifacts or sanitized examples. The workflow does not require credentials, OTPs, cookies, account exports, private customer records, payment data, identity records, private repositories, or raw third-party page bodies.
