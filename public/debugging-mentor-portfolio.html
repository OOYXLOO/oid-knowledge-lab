How I help

Debug the smallest failing boundary first

The most useful live session is not a broad tour through a whole codebase. It is a focused pass over one failing behavior: what should happen, what happens instead, which error or log proves it, and which boundary changes the outcome.

1Reproduce

Start with the smallest visible failure, test, request, log, or screenshot.

2Isolate

Separate payload shape, auth, timing, state, type, and environment assumptions.

3Verify

End with a fix, workaround, or next-step handoff and clear acceptance checks.

Session starter

What to share before a debugging session

Please share:
1. What should happen.
2. What happens instead.
3. The smallest error, failing test, log line, request, or screenshot that proves the issue.
4. The file, function, endpoint, or component where you think the issue starts.

Please remove production passwords, API keys, tokens, cookies, private keys,
payment data, and customer personal data before sharing anything.

Scope boundary
