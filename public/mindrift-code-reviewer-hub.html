Review fit

What this portfolio is meant to prove

The work here is positioned for senior AI code-review tasks rather than general prompt rating. It emphasizes behavior-first review, test design, edge cases, unsafe-input checks, maintainability, corrected implementations, and concise explanations that show why a model response should be accepted, revised, or rejected.

Mindrift-style taskProof path
Review AI-generated codeUse the Python mini lab and evaluator portfolio to inspect behavior contracts, tests, and evidence logs.
Identify issues and subtle bugsFocus on invalid inputs, missing edge cases, overclaimed explanations, weak validation, and brittle branching.
Produce a corrected versionUse a short revise path: describe the failing behavior, fix the implementation, then state tests that prove the change.
Give structured feedbackUse a verdict format with main issues, evidence, suggested fix, tests to add, and residual risk.
