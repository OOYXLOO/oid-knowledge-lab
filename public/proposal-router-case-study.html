Problem

Proposal text needs a target repository before it becomes useful work

When an organization has many repositories, a contributor may describe the right task but file it in the wrong place. A small routing layer can reduce triage work and give automation a cleaner starting point.

Prototype

A compact workspace for proposal routing

AreaPurposeImplementation
Proposal editorCapture the raw task description.Static textarea with live word count.
Ranked matchesShow likely target repositories.Repository profiles, aliases, phrases, keyword overlap, and domain boosts.
ReasonsExplain why each repository appears.Top matched terms and phrases rendered as compact tags.
Parsed hintsPreview routing metadata.Priority, time, price, and target organization heuristics.

Verification

Deterministic tests before embeddings

The first version uses deterministic scoring so the behavior is easy to review. Tests cover payment UI, Telegram transcription, plugin configuration, urgent hints, and token normalization.
