Why it exists

A small, testable proof for Python writing and DevRel pitches

The sample demonstrates a practical OID workflow with real code: parse a public certificate, enumerate extension and usage OIDs, map friendly names, and render a sanitized Markdown handoff. It supports Python-focused article pitches and technical writing samples without relying on private data.

Run it

Minimal command path

python -m pip install cryptography
python certificate_oid_inspector.py path/to/public-certificate.pem --format markdown
python test_certificate_oid_inspector.py

Publication boundary

Safe inputs only

The sample uses public certificate metadata only. It should not be used with private keys, customer certificates, account screenshots, cookies, tokens, payment data, identity records, KYC/tax data, or production logs.
