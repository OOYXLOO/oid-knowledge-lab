What is implemented

  • OID dataset ingestion and public index packaging.
  • Deterministic OID parsing, source policy checks, and coverage reporting.
  • Qwen remediation demo packets and a generated submission pack.
  • Public review pages that do not require private account access.

What is not claimed

  • No claim of a live production customer deployment.
  • No claim of storing private API keys or private console screenshots.
  • No claim that Qwen decides source truth; deterministic checks remain the source boundary.
  • No claim that external actions are taken without human approval.

Evaluator checklist

  • Useful problem: messy OID evidence is hard to review, explain, and remediate safely.
  • Qwen fit: Qwen turns sanitized deterministic findings into readable reviewer handoffs.
  • Safety boundary: source checks are deterministic; generated text remains reviewable.
  • Public proof: demo pages, raw Markdown packs, source code, and generated reports are public.
  • Next step: a live Qwen Cloud run can be added later as a redacted receipt, not as a hidden screenshot.
