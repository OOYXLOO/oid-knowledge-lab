3-minute walkthrough script

OID Knowledge Lab: Qwen Remediation Autopilot

A recording-ready walkthrough for judges: sanitized OID evidence enters deterministic checks, Qwen drafts reviewer notes from that packet, and a human gate stops external action.

Open packet Architecture

0:00-0:25 Problem

OID inventories look harmless until a certificate policy, SNMP/MIB, IAM migration, or registry cleanup forces a team to prove which identifiers are source-backed and which rows need remediation.

0:25-0:55 Deterministic Evidence

The local engine parses sanitized OID rows, checks public IANA PEN evidence, classifies malformed values, and writes JSON/Markdown handoffs before any model output is trusted.

0:55-1:35 Qwen Role

Qwen is positioned as the language and reasoning layer. It summarizes source-grounded evidence, explains uncertain rows, and drafts remediation notes from the provided packet.

1:35-2:05 Human Gate

The workflow stops before vendor contact, customer handoff, ticket creation, registry changes, or public publication until a reviewer approves the next step.

2:05-2:35 Alibaba Entry Point

The repository includes an Alibaba Function Compute handler for a private DashScope-compatible Qwen call. It requires a private environment variable and publishes no secrets.

2:35-3:00 Submission Boundary

Current public proof is source-level and offline-ready. A live deployment claim should only be added after a private run produces a redacted receipt with no API keys, billing data, cookies, tokens, or console screenshots.
