Media Ledger Studio proof

Generated media objects need paired metadata sidecars.

This public proof page shows the storage handoff shape used by Media Ledger Studio: each generated image, video, or audio asset has a Backblaze B2-shaped object record and a JSON sidecar key that can travel with the final media file.

Sidecar body

What the sidecar preserves

FieldPurpose
objectKeyConnects the metadata record to the final media object.
checksumSha256Lets reviewers verify that the reviewed asset is the same stored asset.
provider / model / seedKeeps generation provenance available after storage handoff.
licenseShows whether the output is production-ready or still needs a rights check.
reviewDecision / safetyNotesPreserves the human review outcome beside the media object.

Verification

Where to inspect the implementation

The source snapshot includes the manifest generator, tests, submission pack export, static UI, and audit script. The live adapter can later upload each media object and its sidecar record to a real B2 bucket without changing the reviewer-facing ledger shape.
