Positioning

Review method

I review technical work by turning ambiguous claims into behavior contracts, focused tests, evidence logs, and clear accept / revise / reject decisions.

1 Define expected behavior

Restate the task, reader outcome, or implementation contract before judging the answer.

2 Find risk early

Identify edge cases, unsafe inputs, missing evidence, and places where generated code overclaims.

3 Check before acceptance

Use focused tests, source facts, or reproducible commands before recommending acceptance.

4 Give a verdict

Return an accept, revise, or reject decision with evidence, suggested fix, and residual risk.
