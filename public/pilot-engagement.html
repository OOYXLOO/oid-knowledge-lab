Pilot scope

What a first pass includes

A first pass should be easy to approve, easy to review, and easy to reject if the fit is wrong. The package below is sized for one workflow and one acceptance decision.

1. Intake Confirm the source boundary

Define the allowed public docs, sanitized rows, example thread, or asset list. Exclude credentials, private tickets, payment data, copied third-party page bodies, and account exports.

2. Processing Generate the review packet

Run the relevant local tool, preserve input assumptions, classify unresolved items, and produce a compact handoff for a human reviewer.

3. Acceptance Check whether it reduced review work

The pilot succeeds when the reviewer can see what is verified, what is unsupported, what is blocked, and what follow-up decision is needed.

4. Handoff Deliver reproducible artifacts

Provide Markdown, CSV, JSON, or a static review page plus the command or browser workflow used to produce it.
