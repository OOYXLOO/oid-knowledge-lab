Article fit

Implementation-backed Hasura tutorial

The tutorial would show developers how to build a small review API from a sanitized OID inventory. Hasura is the useful center of the article: it turns Postgres review tables into GraphQL queues, applies role permissions, and keeps mutation scope separate from public reporting.

Proposed outline

Article structure

  1. Why OID inventory cleanup benefits from a review API.
  2. Create Postgres tables for sanitized OID assets and findings.
  3. Track the tables in Hasura and inspect generated GraphQL.
  4. Build open-finding, certificate-row, and owner-queue queries.
  5. Add a restricted mutation for triage updates.
  6. Define safe export rules for public Markdown or CSV handoffs.

Copy-ready pitch

Short application text

I would like to pitch an implementation-backed Hasura tutorial: Build a GraphQL Review API for OID Inventory Cleanup.

The article would show how to model sanitized Object Identifier inventory rows in Postgres, expose reviewer queues with Hasura GraphQL, add role-aware status updates, and publish safe Markdown/CSV summaries without private keys, customer inventories, raw scans, account exports, or copied third-party registry page bodies.
