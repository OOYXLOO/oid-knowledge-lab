Role fit

Evaluator workflow

The evaluator does not treat model output as final code. The workflow starts by defining expected behavior, then checks generated code against tests, unusual inputs, unsafe assumptions, and traceable source facts. The final handoff explains why a response should be accepted, revised, or rejected.

1 Define behavior

Turn a prompt or task into acceptance criteria before judging generated code.

2 Test boundaries

Use focused tests for normal cases, edge cases, invalid inputs, and unsafe data handling.

3 Explain evidence

Keep a concise log of checks, failures, fixes, and remaining assumptions.

Boundary

What this portfolio does not include

The public samples avoid credentials, account exports, private customer data, private platform messages, payment information, tax records, identity documents, copied third-party article bodies, and raw private workspace data. The portfolio is limited to original code, generated artifacts, and review patterns that can be inspected publicly.
