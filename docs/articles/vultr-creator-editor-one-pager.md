# Vultr Creator editor one-pager: OID evidence dashboard on cloud infrastructure

This one-pager supports a Vultr Creator Program application. It is not a finished publication draft.

## Working title

Build a Static OID Evidence Dashboard on Vultr with Node.js and Object Storage

## One-sentence pitch

Show developers how to generate safe JSON and Markdown evidence artifacts with Node.js, publish a static dashboard, and keep credentials, private inventories, and copied third-party page bodies out of the release.

## Why this fits Vultr readers

Vultr readers often want practical infrastructure walkthroughs that are reproducible, cloud-hosted, and useful for real operations. This topic combines a small Node.js data workflow with static hosting, object storage, release checks, and a public proof surface. It can be adapted for compliance evidence, asset inventories, release handoffs, documentation review packs, and lightweight internal dashboards.

## Proposed reader outcome

After reading the tutorial, a developer should be able to:

1. Set up a small Node.js project for safe evidence generation.
2. Convert public or sanitized local input into JSON and Markdown reports.
3. Build a static dashboard from derived artifacts.
4. Add a publish guard that blocks secrets, local-only files, and raw mirrors.
5. Deploy the dashboard to a simple cloud-hosted static surface.
6. Explain which data belongs in public artifacts and which data must remain local.

## Proposed outline

1. Introduction: why evidence dashboards need a publish boundary.
2. Prerequisites: Node.js, npm, Git, and a Vultr account or static hosting target.
3. Create the project structure.
4. Add safe sample input.
5. Generate JSON and Markdown reports.
6. Build the HTML dashboard.
7. Add a manifest and publish guard.
8. Deploy the static output.
9. Verify the site and generated artifacts.
10. Extend the workflow for operational evidence packs.

## Publication boundary

The public sample contains derived reports, safe example inputs, public proof links, and generated dashboard artifacts. It does not publish credentials, cloud account exports, customer inventories, payment data, API tokens, cookies, raw third-party page-body mirrors, contact-level exports, or local-only crawl output.
