# Tuts+ Tutorial Readiness Pack

## Current status

This packet supports a Tuts+ Code tutorial pitch. The public Tuts+ author pages route prospective authors through a tutorial idea form and describe paid tutorial work. Use this as a backup after the current P1 lanes, not as a replacement for Real Python, Vultr, or Draft.dev.

Proposed topic:

```text
Build a Static Evidence Dashboard with Node.js, JSON Reports, and a Publish Guard
```

## Tutorial fit

This proposal fits a practical web-development tutorial because it teaches a complete static-site workflow:

- Generate structured JSON from a safe sample input.
- Render a static HTML dashboard.
- Add a Markdown handoff and dataset manifest.
- Add a publish guard that blocks unsafe generated artifacts.
- Deploy or preview the static output without requiring a private backend.

## Reader promise

By the end of the tutorial, readers should know how to:

- Design a small report format for reviewable developer handoffs.
- Generate JSON and Markdown artifacts with Node.js.
- Render a static dashboard from generated data.
- Add simple tests and release checks.
- Keep credentials, private inventories, account exports, and copied third-party page bodies out of the public package.

## Proposed article flow

1. Create a small Node.js project.
2. Add a safe sample input file.
3. Generate a JSON report.
4. Render a static dashboard page.
5. Add a Markdown summary and manifest.
6. Add a publish guard.
7. Run checks before sharing the static output.
