# Envato Tuts+ Instructor Pack

## Current status

This packet supports an Envato Tuts+ instructor-style submission. The current public Teach page invites authors to teach a course or create a tutorial. The course path asks for at least 30 minutes of video teaching, while the tutorial path can be written or screencast-based.

Use this as a curated backup lane after the strongest P1 writing applications. It is especially useful when a form asks for both technical writing and video-teaching evidence.

## Why this fits

The OID Knowledge Lab project already contains the core proof needed for an instructor application:

- A working Node.js project with tests and generated artifacts.
- A static dashboard that can be inspected without credentials.
- A publish guard that rejects unsafe public artifacts.
- A long-form tutorial sample.
- A reviewer hub that routes editors to the proof links.

## Written tutorial outline

1. Define a small evidence report format.
2. Generate JSON and Markdown outputs with Node.js.
3. Render a static dashboard page.
4. Add a dataset manifest.
5. Add a publish guard for unsafe artifacts.
6. Build the site and run checks.
7. Share the review URL and source repository safely.

## Video course outline

3. Generate JSON and Markdown artifacts.
4. Render the static dashboard.
5. Add tests and a publish guard.
6. Build and inspect the public site.

## Copy-ready application note

I build implementation-backed developer education materials. My strongest sample is a Node.js static evidence dashboard that generates JSON and Markdown reports, renders a public review page, and adds a publish guard to keep private inputs, credentials, account exports, and copied third-party page bodies out of the public package. The same project can support a written tutorial or a short video course because the repository includes runnable scripts, tests, generated artifacts, a live dashboard, and reviewer proof links.

## Submission boundary

- No credentials.
- No private account exports.
- No private customer inventories.
- No payment, tax, or identity data.
- No copied full third-party page bodies.
- No claim that a sample has already been accepted by an editor.
- If an idea is accepted, write or record the final tutorial freshly for Envato Tuts+' current author guidelines, technical review, originality requirements, and editor feedback.
