# Vultr Creator Readiness Pack

## Creator fit

This proposal is a good fit for the Vultr Creator Program because it turns a small, realistic developer workflow into a reproducible cloud tutorial:

- Generate JSON and Markdown review artifacts with Node.js.
- Publish a static evidence dashboard.
- Keep private inventories, credentials, copied page bodies, and account exports out of public output.
- Run a release guard before sharing the dashboard.

The proposed tutorial is:

```text
Build a Static OID Evidence Dashboard on Vultr with Node.js and Object Storage
```

## Reader promise

By the end of the tutorial, a reader should know how to:

- Shape a small public or sanitized dataset into a reviewable JSON report.
- Generate a static dashboard from that report.
- Add a manifest that explains which sources are public, local-only, or authorization-gated.
- Publish the static output on cloud infrastructure.
- Run a final guard that prevents secrets, private data, or copied third-party page bodies from being released.

## Proposed tutorial flow

1. Start with a sanitized CSV or public registry index.
2. Generate a normalized JSON report.
3. Render a static HTML dashboard.
4. Add a Markdown handoff and dataset manifest.
5. Prepare the static site for Vultr-hosted publication.
6. Run a publish guard.
7. Verify the public URL and explain the safe-data boundary.

## Why this is useful for Vultr readers

The tutorial shows a practical pattern for developers who do not need a large application stack:

- No database required for the first version.
- No always-on private backend.
- Clear public/private data split.
- Useful for internal evidence dashboards, release review packets, small documentation portals, and operational handoffs.
