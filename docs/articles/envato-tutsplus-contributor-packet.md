# Envato Tuts+ Code Contributor Packet

## Profile Summary

I write practical developer tutorials that turn small, verifiable tools into reusable workflows. My current examples focus on browser-only JavaScript, source-boundary review, asset provenance, and evidence-first technical documentation.

## Tutorial Idea 1

Title: Build a Local-First Provenance Ledger for AI-Generated Media Assets

Pitch: AI-generated images, diagrams, audio, and video assets are easy to create but hard to hand off responsibly. This tutorial shows how to build a browser-only JavaScript ledger with asset records, review status, model notes, search filters, live summary metrics, and JSON or Markdown export.

Reader outcome:
- Understand a minimal provenance data model.
- Build a dependency-free HTML and JavaScript review app.
- Export a delivery sheet that can be shared with editors, reviewers, or clients.

Proof:

## Tutorial Idea 2

Title: Build a Browser-Only Evidence Dashboard for Public Registry Data

Pitch: Many developer workflows depend on public registries, but teams often copy identifiers into docs, agents, or dashboards without keeping source links and review notes. This tutorial shows how to build a browser-only evidence dashboard that loads structured public records, supports search, links source pages, and creates a review-friendly handoff page.

Reader outcome:
- Convert registry-like records into a searchable static dataset.
- Build a lightweight dashboard with source links and review notes.
- Keep private data and credentials out of a public demo.
