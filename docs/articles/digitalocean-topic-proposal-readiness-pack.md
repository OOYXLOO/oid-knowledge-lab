## Current status

The public DigitalOcean writing page has historically described paid tutorials, but it has also shown pause language. Submit only if the current entry flow presents a clear topic or proposal form.

Proposed topic:

```text
Build a Static Evidence Dashboard with Node.js and a Publish Guard
```

## Tutorial fit

This proposal fits a practical developer tutorial because it teaches a complete, reproducible workflow:

- Generate JSON and Markdown artifacts with Node.js.
- Build a static HTML dashboard from those artifacts.
- Add a dataset manifest and source-boundary notes.
- Run a publish guard before sharing the output.
- Keep credentials, account exports, private inventories, and copied third-party page bodies out of the public site.

## Reader promise

By the end of the tutorial, a reader should know how to:

- Start from safe public or sanitized input.
- Generate a reviewable report.
- Build a static dashboard that can be hosted without a database.
- Include a manifest that explains what is public, local-only, or authorization-gated.
- Prevent unsafe generated artifacts from being published.

## Proposed article flow

1. Set up a small Node.js project.
2. Define a safe sample input file.
3. Generate JSON and Markdown review artifacts.
4. Render the static dashboard.
5. Add a dataset manifest.
6. Add a publish guard.
7. Run tests and build checks.
8. Publish the static output and verify the public URL.
