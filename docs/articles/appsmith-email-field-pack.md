## Short pitch

```text
I would like to pitch an original Appsmith Writers Program article:

Build a Client-Safe OID Inventory Review App with Appsmith

The article would show how to build an Appsmith internal review app for a sanitized OID inventory. Reviewers could import a safe CSV, inspect public OID evidence, classify rows, mark status, and export a remediation queue without exposing credentials, customer data, raw private exports, or copied third-party page bodies.
```

## Why it fits Appsmith

```text
- It is an internal-tool tutorial for a realistic governance and review workflow.
- Appsmith is the UI layer for reviewer actions, filters, status updates, evidence panels, and handoff output.
- The topic can become a forkable template or tutorial, not only a conceptual article.
- It fits developers and technical teams building secure internal apps.
```

## Proposed outline

```text
1. Why OID inventory reviews need a client-safe internal workflow.
2. Define a sanitized CSV input shape.
3. Create Appsmith pages: Intake, Review Queue, Evidence Panel, and Handoff.
4. Add tables for inventory rows and public evidence summaries.
5. Add filters for invalid syntax, unknown enterprise numbers, known rows, subtree-only matches, and unresolved rows.
6. Use a JSObject to format a copyable Markdown remediation queue.
7. Add validation checks that keep private identifiers, credentials, account exports, and copied third-party page bodies out of the handoff.
8. Link the Appsmith app to a static public proof page and dataset manifest.
9. Run acceptance checks before sharing results.
```
