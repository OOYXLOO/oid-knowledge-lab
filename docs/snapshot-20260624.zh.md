## 合规边界

OID-base 的站点条款限制了复制和下载站内数据的范围。当前公开包只保存 sitemap 级目录、公开来源链接、统计信息、IANA 开放注册表数据和分析结果，不保存 OID-base 页面正文或原始镜像。

如果以后拿到站点所有者的明确授权，可以使用授权模式在本地执行完整正文采集：

```powershell
$env:OID_BASE_FULL_CRAWL_AUTHORIZED = "1"
node src/cli.js crawl --authorized-full --authorization-note "authorization reference" --delay-ms 1500 --out data/full
```

授权采集结果默认仍然留在 Git 忽略路径中，除非授权明确允许公开再分发。

## 项目价值方向

这个项目适合继续发展成 OID 目录索引、企业号检索、OID 资产清单质量检查、OID 覆盖分析和公开 registry 数据分析工具。

更有价值的方向是让使用者拿自己的 OID/MIB/PKI 资产清单做本地分析，自动识别无效 OID、未知企业号、缺失证据链、内部命名空间和可公开佐证的 registry 映射。
